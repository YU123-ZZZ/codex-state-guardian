# ChatGPT(Codex) 设置和聊天记录备份

这是一个给 Windows 用的 ChatGPT(Codex) 本地备份恢复脚本。

最后更新：2026-07-13

它的作用很简单：把 ChatGPT(Codex) 的设置、插件状态、聊天记录、聊天索引和聊天列表状态保存下来。  
如果后面出现设置丢失、插件异常、聊天记录不全、聊天列表不对等情况，可以用于备份恢复。

## 最重要的事

这个脚本不是简单覆盖文件。

- 备份时：完整保存设置、聊天、插件、模型缓存和 `pets`
- 恢复时：自动识别 API 地址、API key、token、secret、password 等连接配置
- 旧备份不会把当前机器正在用的新 API 地址和 key 顶掉
- 旧备份没有的新连接字段，也会尽量保留在当前 `config.toml` 里
- 模型、模型提供商、推理强度等普通模型配置，会按备份恢复

人话：  
恢复旧备份时，模型可以跟着备份走；API 地址和密钥优先保留你现在能用的。

## 适合什么情况

- 想提前保存一份当前正常状态
- 担心更新、异常退出或误操作后丢设置
- 想把聊天记录和聊天状态一起备份下来
- 想在出问题后快速恢复
- 换过 API 地址、base_url 或 key，怕恢复旧备份后又被改回旧配置

## 有哪些功能

- 保存 ChatGPT(Codex) 设置
- 完整保存 `config.toml`
- 恢复设置时自动保护当前 API 地址和密钥
- 保存当前模型选择和模型目录缓存
- 保存更新后的 Browser、Computer Use 状态等
- 保存 ChatGPT(Codex) 宠物资源（`pets`）
- 保存插件状态
- 保存聊天记录
- 保存聊天索引和聊天列表状态
- 可以选择完整备份、聊天备份或设置备份
- 恢复最新一个备份，并按当时选择的范围恢复
- 恢复指定备份
- 恢复前自动生成一份“恢复前备份”，方便回退
- 默认最多保留 10 个备份
- 至少保留最新 2 个健康备份
- 打开菜单时自动巡检当前状态，并直接提示下一步该怎么做

## 会备份哪些内容

默认备份的是当前 Windows 用户目录下的 ChatGPT(Codex) 数据：

```text
%USERPROFILE%\.codex
```

主要包括这些内容：

```text
config.toml
cc-switch-model-catalog.json
browser
computer-use
cache\computer-use
pets
.codex-global-state.json
session_index.jsonl
sessions
archived_sessions
state_5.sqlite
state_5.sqlite-wal
state_5.sqlite-shm
```

人话：

- 设置
- 当前模型、模型提供商、模型目录缓存
- 插件状态
- ChatGPT(Codex) 宠物资源和配置
- 聊天记录
- 聊天索引
- 聊天列表相关状态

## 三种备份范围

新建备份时，脚本会先问这次想保存什么：

- `完整备份`：设置、聊天记录和相关状态一起保存
- `聊天备份`：只保存聊天记录、聊天索引、SQLite 状态和归档聊天
- `设置备份`：只保存设置、模型缓存、Browser、Computer Use、插件状态和 `pets`

恢复时会自动沿用这份备份的范围。  
例如，恢复“聊天备份”不会覆盖当前设置；恢复“设置备份”不会改动当前聊天记录。

## 备份会保存到哪里

备份默认保存在脚本同目录下的：

```text
chatgpt-codex-state-backups
```

如果这个文件夹不存在，脚本会自动创建，不需要手动新建。

## 第一次怎么用

1. 把 `chatgpt-codex-state-guardian.cmd` 放到一个固定文件夹里。
2. 双击运行脚本。
3. 第一次建议按：

```text
1. 备份
```

4. 范围选 `完整备份`。

脚本会先检查。正常才保存。

## 菜单说明

双击打开后，顶部会先巡检：

- ChatGPT(Codex) 数据目录是否存在
- ChatGPT(Codex) 是否正在运行
- 当前状态是否适合备份
- 有没有可用健康备份
- 最近的完整备份、聊天备份、设置备份分别是哪一份
- 备份数量是否已经到上限
- `pets`、Browser、Computer Use 这些常见配置是否能检测到

有问题会直接提示下一步：

- 先关闭 ChatGPT(Codex) 再恢复
- 当前状态不适合完整备份，先查看状态或恢复旧备份
- 还没有健康备份，建议先建立第一份完整备份
- Browser 或 Computer Use 配置缺失时，提示先重启 ChatGPT(Codex) 或执行修复

```text
1. 备份：检查正常后保存
2. 状态：查看本机文件
3. 列表：查看备份文件夹
4. 恢复：使用最新备份
5. 指定恢复：输入备份名
6. 强制备份：不管重复也保存
0. 退出
```

平时最常用的是：

```text
1 备份
3 找备份文件夹
4 恢复最新备份
5 恢复指定备份
```

## 怎么保存备份

1. 双击 `chatgpt-codex-state-guardian.cmd`
2. 选择 `1. 备份`
3. 选择 `完整备份`、`聊天备份` 或 `设置备份`
4. 等脚本检查并保存完成

状态正常才会保存。  
如果内容和最近同类备份一样，脚本不会重复保存。

备份名称会直接写清范围：

```text
完整备份-20260701-101046
聊天备份-20260701-101110
设置备份-20260701-101135
```

如果你想不管是否重复，都强制保存一份新备份，可以选择：

```text
6. 强制备份
```

## 保存时脚本会做什么

- 检查当前状态是不是基本正常
- 按你选择的范围决定要复制哪些文件
- 读取当前模型、模型提供商和模型目录
- 检查是不是和最近一份同范围备份完全一样
- 如果不一样，就创建新的备份
- 如果 ChatGPT(Codex) 还开着，会建议先关闭
- 也可以在线备份，但必须通过校验
- 校验失败会自动删除临时结果
- 只有严格校验通过的结果才会算作正式备份

这样可以避免“看起来备份了，其实没备完整”。

如果脚本检测到 ChatGPT(Codex) 正在运行，会给两个选择：

- 推荐：先关闭 ChatGPT(Codex)，关完后再继续备份
- 允许：继续在线备份。失败会自动删除结果

## 怎么查看当前状态

运行脚本后选择：

```text
2. 查看当前状态
```

能看到：

- 当前 ChatGPT(Codex) 数据目录
- 当前备份目录完整路径
- 当前模型和模型目录数量
- 已启用插件数量
- 聊天相关文件数量
- 最新备份位置

## 怎么查看已有备份

运行脚本后选择：

```text
3. 列出备份
```

能看到每一份备份的：

- 名称
- 类型
- 备份范围
- 备份里的模型
- 插件数量
- 宠物文件数量
- 聊天文件数量
- 保存时间
- 每份备份的完整路径

大小会尽量显示成人容易读的格式，比如 `97.70 MB`，不用自己换算字节。

## 怎么恢复最新一个备份

出问题时这样恢复：

1. 先关闭 ChatGPT(Codex)
2. 双击 `chatgpt-codex-state-guardian.cmd`
3. 选择 `4. 恢复`
4. 看清脚本显示的备份信息
5. 输入：

```text
确认
```

只有输入 `确认` 才会改文件。  
直接回车就是取消。

确认前会显示路径和范围：

- 聊天备份：只恢复聊天
- 设置备份：只恢复设置
- 完整备份：恢复设置 + 聊天

恢复 `config.toml` 时，模型、模型提供商、推理强度等模型配置会跟随备份同步，例如备份里是新模型，恢复后就会切到备份里的新模型。

恢复时还会应用 `cc-switch-model-catalog.json`，这个文件保存的是本地模型目录缓存。它能减少“配置里想用新模型，但恢复后模型列表还是旧的”的情况。

设置备份会完整保存当时的 `config.toml`，不是只保存一部分设置。

恢复 `config.toml` 时会自动识别连接配置。  
旧备份不会覆盖当前机器正在用的 API 地址和 key。  
旧备份里没有、但当前新配置里有的连接字段，也会尽量保留。  
会保护这些字段：

```text
base_url
api_url
endpoint
host
url
proxy
key
token
secret
password
credential
```

简单说：

- 模型配置按备份恢复
- API 地址、base_url、endpoint、host、API key、token、secret、password 等保留当前值
- 新配置里已有但旧备份没有的连接字段，不会轻易丢

如果你想确认模型有没有保存进去，可以先选择：

```text
3. 列出备份
```

列表里会显示每份备份保存的模型和模型目录数量。

## 怎么恢复指定备份

1. 先关闭 ChatGPT(Codex)
2. 双击 `chatgpt-codex-state-guardian.cmd`
3. 先选择 `3. 列出备份`
4. 记下想恢复的完整备份文件夹名
5. 选择 `5. 指定恢复`
6. 输入完整备份文件夹名
7. 输入：

```text
确认
```

## 恢复前会不会先留一份现场

会。

每次恢复之前，脚本都会先自动保存一份“恢复前备份”，方便你后悔时回退。

名称类似：

```text
恢复前备份-20260701-101000
```

## 恢复后如果发现不对，怎么回退

如果恢复后发现恢复错了，或者恢复结果不是你想要的，可以再恢复刚才那份“恢复前备份”：

1. 关闭 ChatGPT(Codex)
2. 双击 `chatgpt-codex-state-guardian.cmd`
3. 选择 `3. 列出备份`
4. 找到最新的 `恢复前备份-*`
5. 选择 `5. 恢复指定备份`
6. 输入那份恢复前备份的完整文件夹名
7. 输入：

```text
确认
```

这样就相当于把恢复前的状态再恢复回来。

## 如果 ChatGPT(Codex) 还开着，能不能恢复

不建议，也不应该。

如果恢复时脚本检测到 ChatGPT(Codex) 还在运行，会直接停止恢复，避免刚恢复完又被正在运行的 ChatGPT(Codex) 覆盖回去。

所以恢复前最好先把 ChatGPT(Codex) 完全关闭。

## 备份数量怎么保留

- 默认最多保留 10 个备份
- 超过 10 个时，会自动删除较旧的备份
- 脚本会尽量保留最新 2 个健康备份

这样做是为了避免恢复前备份太多，把正常健康备份全挤掉。

## 最方便用法

平时就用这一种：

1. 双击 `chatgpt-codex-state-guardian.cmd`
2. 按菜单编号操作
3. 恢复时还是要输入 `确认`；直接回车会取消

## 使用时的注意事项

- 恢复前建议先关闭 ChatGPT(Codex)
- 保存时如果数据正在变化，脚本会自动重试几次
- 如果还是不稳定，建议先关闭 ChatGPT(Codex) 再保存
- 如果刚切换了新模型，建议先正常退出 ChatGPT(Codex)，再运行脚本保存一次
- 不输入 `确认`，脚本不会覆盖当前文件
- 备份里可能包含个人设置和聊天记录
- 不要手动修改备份文件夹里的内容

## 一句话理解

平时用它保存一份当前正常状态；  
出问题时，用它把之前保存的状态恢复回来。
