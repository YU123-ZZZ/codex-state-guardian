# Codex 设置和聊天记录备份

这是一个给 Windows 用的 Codex 本地备份恢复脚本。

它的作用很简单：把 Codex 的设置、插件状态、聊天记录、聊天索引和聊天列表状态保存下来。  
如果后面出现设置丢失、插件异常、聊天记录不全、聊天列表不对等情况，可以用备份恢复。

## 适合什么情况

- 想提前保存一份当前正常状态
- 担心更新、异常退出或误操作后丢设置
- 想把聊天记录和聊天状态一起备份下来
- 想在出问题后快速恢复

## 有哪些功能

- 保存 Codex 设置
- 保存插件状态
- 保存聊天记录
- 保存聊天索引和聊天列表状态
- 恢复最新健康备份
- 恢复指定备份
- 恢复前自动生成一份“恢复前备份”，方便回退
- 默认最多保留 5 个备份
- 至少保留最新 2 个健康备份

## 文件说明

```text
codex-state-guardian.cmd  主脚本
README.md                使用说明
```

## 会备份哪些内容

默认备份的是当前 Windows 用户目录下的 Codex 数据：

```text
%USERPROFILE%\.codex
```

主要包括这些内容：

```text
config.toml
.codex-global-state.json
session_index.jsonl
sessions
archived_sessions
state_5.sqlite
state_5.sqlite-wal
state_5.sqlite-shm
```

简单说，就是：

- 设置
- 插件状态
- 聊天记录
- 聊天索引
- 聊天列表相关状态

## 备份会保存到哪里

备份默认保存在脚本同目录下的：

```text
codex-state-backups
```

如果这个文件夹不存在，脚本会自动创建，不需要手动新建。

## 第一次怎么用

1. 把 `codex-state-guardian.cmd` 放到一个固定文件夹里。
2. 双击运行脚本。
3. 第一次建议直接选择：

```text
1. 检查并保存健康备份
```

这个选项的意思是：检查当前 Codex 状态是否正常，正常就保存一份备份。

## 菜单说明

```text
1. 检查并保存健康备份
2. 查看当前状态
3. 列出备份
4. 恢复最新健康备份
5. 恢复指定备份
6. 强制保存一份健康备份
0. 退出
```

平时最常用的是：

```text
1 保存当前正常状态
3 查看已有备份
4 恢复最新健康备份
5 恢复指定备份
```

## 怎么保存备份

1. 双击 `codex-state-guardian.cmd`
2. 选择 `1. 检查并保存健康备份`
3. 等脚本检查并保存完成

如果当前状态正常，而且和最新健康备份不同，脚本就会创建一份新的健康备份。

备份名称类似：

```text
健康备份-20260701-101046
```

如果你想不管是否重复，都强制保存一份新备份，可以选择：

```text
6. 强制保存一份健康备份
```

## 保存时脚本会做什么

- 检查当前状态是不是基本正常
- 检查是不是和最新健康备份完全一样
- 如果不一样，就创建新的健康备份
- 如果保存时正好遇到 Codex 正在写入聊天数据，脚本会自动重试
- 如果连续几次都不稳定，会提示先关闭 Codex 后再保存

这一步是为了尽量避免“看起来保存了，但聊天状态没有保存完整”的问题。

## 怎么查看当前状态

运行脚本后选择：

```text
2. 查看当前状态
```

可以看到：

- 当前 Codex 数据目录
- 当前备份目录
- 已启用插件数量
- 聊天相关文件数量
- 最新健康备份位置

## 怎么查看已有备份

运行脚本后选择：

```text
3. 列出备份
```

可以看到每一份备份的：

- 名称
- 类型
- 插件数量
- 聊天文件数量
- 保存时间

## 怎么恢复最新健康备份

如果 Codex 的设置、插件、聊天记录或聊天列表出现问题，可以这样恢复：

1. 先关闭 Codex
2. 双击 `codex-state-guardian.cmd`
3. 选择 `4. 恢复最新健康备份`
4. 看清脚本显示的备份信息
5. 输入：

```text
确认
```

只有输入 `确认`，脚本才会真正覆盖当前文件。  
如果直接回车，或者输入别的内容，恢复会取消。

恢复 `config.toml` 时，脚本默认会保留你当前正在使用的模型、模型提供商和相关密钥配置，避免恢复旧备份后把模型和密钥也一起回滚。

## 怎么恢复指定备份

1. 先关闭 Codex
2. 双击 `codex-state-guardian.cmd`
3. 先选择 `3. 列出备份`
4. 记下想恢复的完整备份文件夹名
5. 选择 `5. 恢复指定备份`
6. 输入完整备份文件夹名
7. 输入：

```text
确认
```

## 恢复前会不会先留一份现场

会。

每次恢复之前，脚本都会先自动保存一份“恢复前备份”，方便你后悔时回退。

名称类似：

```text
恢复前备份-20260701-101000
```

## 恢复后如果发现不对，怎么回退

如果恢复后发现恢复错了，或者恢复结果不是你想要的，可以再恢复刚才那份“恢复前备份”：

1. 关闭 Codex
2. 双击 `codex-state-guardian.cmd`
3. 选择 `3. 列出备份`
4. 找到最新的 `恢复前备份-*`
5. 选择 `5. 恢复指定备份`
6. 输入那份恢复前备份的完整文件夹名
7. 输入：

```text
确认
```

这样就相当于把恢复前的状态再恢复回来。

## 如果 Codex 还开着，能不能恢复

不建议，也不应该。

如果恢复时脚本检测到 Codex 还在运行，会直接停止恢复，避免刚恢复完又被正在运行的 Codex 覆盖回去。

所以恢复前最好先把 Codex 完全关闭。

## 备份数量怎么保留

- 默认最多保留 5 个备份
- 超过 5 个时，会自动删除较旧的备份
- 脚本会尽量保留最新 2 个健康备份

这样做是为了避免恢复前备份太多，把正常健康备份全挤掉。

## 命令行用法

除了双击运行，也可以直接在命令行里用：

```cmd
codex-state-guardian.cmd 状态
codex-state-guardian.cmd 检查
codex-state-guardian.cmd 列表
codex-state-guardian.cmd 恢复
codex-state-guardian.cmd 指定恢复 健康备份-20260701-101046
codex-state-guardian.cmd 检查 -强制
```

旧英文命令也可以使用：

```cmd
codex-state-guardian.cmd check
codex-state-guardian.cmd status
codex-state-guardian.cmd list
codex-state-guardian.cmd restore
```

## 使用时的注意事项

- 恢复前建议先关闭 Codex
- 保存时如果数据正在变化，脚本会自动重试几次
- 如果还是不稳定，建议先关闭 Codex 再保存
- 不输入 `确认`，脚本不会覆盖当前文件
- 备份里可能包含个人设置和聊天记录
- 不要手动修改备份文件夹里的内容

## 一句话理解

平时用它保存一份当前正常状态；  
出问题时，用它把之前保存的状态恢复回来。
