# ChatGPT(Codex) 备份恢复守护

这是一个给 Windows 用的 ChatGPT(Codex) 本地备份恢复脚本。双击脚本就能用，先看状态，再备份或恢复。

最后更新：2026-07-21

它的作用很简单：把 ChatGPT(Codex) 的设置、插件状态、聊天记录、聊天索引和聊天列表状态保存下来。出问题时，再把可用状态恢复回来。

## 简单介绍

这个脚本不是简单覆盖文件。

- 备份时：完整保存设置、聊天、插件、模型缓存和 `pets`
- 恢复时：自动识别 API 地址、API key、token、secret、password 等连接配置
- 旧备份不会把当前机器正在用的新 API 地址和 key 顶掉
- 旧备份没有的新连接字段，也会尽量保留在当前 `config.toml` 里
- 当前厂商和备份厂商相同：模型、推理强度等普通模型设置按备份恢复
- 当前厂商和备份厂商不同：保留当前厂商、当前模型和当前厂商配置，不会从 DeepSeek 等新厂商退回旧厂商
- 当前还没有设置厂商：允许采用备份里的厂商和模型

人话：同一家厂商，模型可以跟着备份走；换了厂商，就保护你现在能用的厂商、模型、API 地址和密钥。

## 有哪些功能

- 三种备份范围：完整备份、聊天备份、设置备份
- 打开后自动巡检当前状态；发现问题会直接说明哪里不正常、下一步该做什么
- 备份时再次检查当前状态
- ChatGPT(Codex) 开着时，先推荐你关闭；也允许在线备份，但必须通过严格校验
- 在线备份如果校验失败，会自动删除失败结果，不会算正式备份
- 如果内容和已有备份重复，会显示重复的文件夹名称和完整路径，不会悄悄跳过
- 恢复时自动按备份范围恢复
- 恢复前自动保存一份“恢复前备份”
- 自动保护当前机器正在使用的 API 地址、base_url、endpoint、host、key、token、secret、password、credential 等连接配置
- 自动识别安装来源：微软商店版 / 安装包版
- 首页自动显示程序安装来源、安装位置和启动入口
- 首页和状态页会分开显示“软件访问入口”“真实数据位置”“实际数据占用”，不会再把 C 盘兼容入口误认为真实数据
- 一键把 ChatGPT(Codex) 在 C 盘的实际占用转到 D 盘
- 微软商店版会尝试通过 Windows 官方移动机制把程序本体转到 D 盘，不会硬搬 `WindowsApps`
- 本地数据转移后自动创建或更新桌面快捷方式
- 也可以单独执行编号 `8`，随时添加或更新桌面快捷方式
- 加载和转移过程使用普通文本进度条，每条进度单独占一行，不会覆盖或挤乱其他文字
- 默认最多保留 10 个备份，并尽量保留最新 2 个健康备份

```text
browser

```

## 三种备份范围

- `完整备份`：设置、聊天记录和相关状态一起保存
- `聊天备份`：只保存聊天记录、聊天索引、SQLite 状态和归档聊天
- `设置备份`：完整保存全部可恢复设置，包括语言、模型、推理强度、快捷键、用户规则、skills、插件状态、Browser、Computer Use 和 `pets`

恢复时会自动沿用这份备份的范围。

语言设置也在设置备份里。脚本会完整保存 `config.toml`，其中包括：

```text
features.enable_i18n
desktop.localeOverride
```

登录凭据、临时缓存、运行日志不属于普通设置，不会冒充“设置内容”覆盖当前机器。

## 最方便用法

1. 双击 `chatgpt-codex-backup-restore-guardian.cmd`
2. 看首页巡检结果
3. 按菜单编号操作

菜单就是这几个：

```text
1. 检查当前状态，正常时保存一份新的备份
2. 查看这次会参与备份或恢复的本地状态
3. 查看已经保存过的备份文件夹
4. 恢复最新的一份可用备份
5. 按名称恢复指定备份
6. 不管是否重复，都新建一份备份
7. 一键把 ChatGPT(Codex) 的 C 盘占用转到 D 盘
8. 添加或更新桌面快捷方式
0. 退出
```

恢复时仍然要输入 `确认`，直接回车就是取消。

如果内容没有变化，脚本会告诉你“这次没有新建备份”，并显示它和哪个备份文件夹重复，方便直接找到原来的文件。

## 运行中备份怎么处理

如果脚本发现 ChatGPT(Codex) 还开着，会给两个选择：

- 推荐：先关闭 ChatGPT(Codex)，再继续备份
- 允许：继续在线备份

但在线备份不是直接算成功。它必须通过严格校验；如果校验失败，结果会自动删除，也不会算正式备份。

## 安装来源怎么区分

脚本会尽量识别当前是下面哪一种：

- `微软商店版`
- `安装包版`

首页巡检和状态页会直接显示安装来源与程序目录，不需要再单独放一个重复菜单。

### 微软商店版

- 程序一般在 `WindowsApps` 下
- 启动走商店应用入口
- 不要手动搬 `WindowsApps` 文件夹
- 脚本会先把本地数据的真实占用转到 D 盘
- 脚本会尝试调用 Windows 官方 Appx 移动功能，把程序本体转到 D 盘
- 如果系统要求更高权限，会明确提示用管理员身份重开脚本，或使用“设置 > 应用 > 已安装的应用 > 移动”

### 安装包版

- 程序一般是普通 `exe` 安装目录
- 启动走 `exe`
- 这个脚本会识别程序目录、CLI 路径和 Computer Use 路径

## 一键把 C 盘占用转到 D 盘

执行编号 `7` 后，脚本会尽量把能安全移动的实际占用转到 D 盘。

本地数据会这样处理：

1. 先做一份“转移前完整备份”
2. 把整个 `.codex` 本地数据复制到 D 盘
3. 自动校验复制结果
4. 把原来的 C 盘目录切成指向 D 盘的链接
5. 再校验新链接
6. 只有全部校验成功，才删除 C 盘旧数据副本并释放空间
7. 自动创建或更新桌面快捷方式

人话：软件表面上仍访问 `C:\Users\你的用户名\.codex`，但这个位置只剩一个很小的兼容链接，真实设置、聊天和运行数据位于 `D:\ChatGPT-Codex-Data\.codex`。

转移完成后，首页和状态页会明确显示：

```text
软件访问入口：C:\Users\你的用户名\.codex
真实数据位置：D:\ChatGPT-Codex-Data\.codex
实际数据占用：已在 D 盘；C 盘只是兼容入口，不保存实际数据
```

这里的 C 盘入口可以理解成“文件夹快捷方式”，但它实际上是 Windows 的文件夹连接（Junction）。不要删除这个入口，否则 ChatGPT(Codex) 会找不到已经转到 D 盘的数据。

程序本体按安装来源处理：

- 微软商店版：使用 Windows 官方 `Move-AppxPackage` 机制尝试移动到 D 盘，不直接复制 `WindowsApps`
- 安装包版：本地数据会转到 D 盘；程序本体应通过安装包重新安装到 D 盘，脚本不会强搬导致更新器或卸载器损坏

## 转盘后为什么还能正常打开

因为脚本保留原来的兼容入口，并按照安装来源保留正确的启动方式。

- 微软商店版：程序仍然走商店启动入口；即使程序本体移动到 D 盘，启动标识不会改变
- 安装包版：程序仍然走原来的 `exe`
- 转移完成后脚本会自动创建或更新桌面快捷方式，方便直接打开

## 单独添加桌面快捷方式

执行编号 `8`，脚本会自动识别当前是微软商店版还是安装包版，然后在桌面创建或更新：

```text
ChatGPT(Codex).lnk
```

- 微软商店版：快捷方式使用系统商店应用入口，不会直接指向受保护的 `WindowsApps`
- 安装包版：快捷方式直接指向已经识别到的程序 `exe`
- 快捷方式已经存在时会安全更新，不会重复创建多个同名文件

## 首页怎么看最新备份

首页会分别显示：

```text
最新完整备份：
最新聊天备份：
最新设置备份：
```

备份根目录仍会显示完整路径，按编号 `3` 可以查看每个备份文件夹的完整路径。

## 恢复时怎么保护 API 地址和 key

设置备份会完整保存当时的 `config.toml`，不是只保存一部分设置。

但恢复时不是简单覆盖。当前机器正在使用的连接配置会优先保留，例如：

```text
base_url
api_url
endpoint
host
url
proxy
key
token
secret
password
credential
```

简单说：

- 备份时会完整保存所有设置，包括语言、模型、模型厂商和厂商区块，不会少保存
- 当前厂商与备份厂商相同：模型和推理参数按备份恢复
- 当前厂商与备份厂商不同：保留当前厂商、当前模型和当前厂商区块；备份里的旧厂商区块不会重新写入
- 当前没有明确设置厂商：允许使用备份里的厂商和模型
- API 地址、base_url、endpoint、host、key、token、secret、password 等保留当前值
- 当前新配置里有、旧备份里没有的连接字段，也尽量保留

例如：当前正在使用 DeepSeek，旧备份使用的是 OpenAI 或自定义厂商。恢复后仍然使用当前 DeepSeek 和当前可用的 DeepSeek 地址、key；语言、界面、快捷键、规则等其他普通设置仍按备份恢复。

## 备份数量怎么保留

- 默认最多保留 10 个备份
- 超过 10 个时，会自动删除较旧备份
- 会尽量保留最新 2 个健康备份

## 适合什么时候用

- 想先保存一份当前正常状态
- 怕更新或异常后设置丢失
- 怕恢复旧备份时把新 API 地址和 key 改回去
- 想把聊天和设置分开备份
- 想把 ChatGPT(Codex) 在 C 盘的实际占用转到 D 盘
